# 商城图片处理规范

## 1. 适用范围

本文档适用于商城模块相关图片字段，包括但不限于：

- 商品分类图片 `picUrl`
- 商品主图列表 `picUrls`
- SKU 图片 `picUrl`
- 订单、购物车、售后等商城快照中的商品图片字段

## 2. 总体原则

商城图片地址统一采用“数据库和后端保存相对路径，前端页面按运行环境补全访问前缀”的处理方式。

统一约定如下：

- 数据库存储相对路径，例如 `/profile/upload/2026/04/10/xxx.png`
- 后端接口返回相对路径，不在后端拼接域名或网关前缀
- 管理端页面根据 `VITE_APP_BASE_API` 补全图片访问地址
- 小程序页面根据 `config/env.js` 中的 `basePath` 补全图片访问地址
- 页面禁止直接假设图片字段一定是完整 `http/https` 地址

这样处理的好处是：

- 开发、测试、生产环境切换时不需要改数据库数据
- 避免把 `localhost`、内网地址、临时域名写入业务数据
- 后端返回结构更稳定，前端可以按当前运行环境自动适配

## 3. 字段约定

### 3.1 单图字段

单图字段统一使用字符串类型，例如：

- `picUrl`
- `avatarUrl`

推荐值：

```text
/profile/upload/2026/04/10/example.png
```

### 3.2 多图字段

多图字段统一使用数组语义，当前商城商品主图字段为 `picUrls`。

前端和后端需要兼容以下几种输入形态：

- 数组：`["/profile/a.png","/profile/b.png"]`
- JSON 字符串：`["/profile/a.png","/profile/b.png"]`
- 逗号分隔字符串：`/profile/a.png,/profile/b.png`

页面展示时统一先规范成数组，再按场景使用：

- 列表页：只展示第一张图
- 详情页：展示完整图片列表
- SKU 默认图：优先取 SKU 图片，没有时回退到商品主图第一张

## 4. 管理端处理规范

### 4.1 公共工具

管理端商城模块统一使用：

- `joolun-wx-ui/src/utils/mall-image.js`

当前提供的方法：

- `parseMallImageList(picUrls)`
- `resolveMallImageUrl(picUrl, fallback)`
- `normalizeMallImageList(picUrls)`
- `getMallFirstImageValue(picUrls, fallback)`
- `getMallFirstImageUrl(picUrls, fallback)`

### 4.2 使用要求

管理端页面禁止直接写下面这类逻辑：

```js
import.meta.env.VITE_APP_BASE_API + row.picUrl
```

也禁止直接在模板中写：

```vue
:src="row.picUrls[0]"
```

统一改为调用公共工具：

```js
parseMallImageList(row.picUrls)
resolveMallImageUrl(row.picUrl)
getMallFirstImageValue(row.picUrls)
getMallFirstImageUrl(row.picUrls)
```

其中：

- 表单赋值、保存提交、默认值回填时使用原始值方法
- 页面展示、预览、列表缩略图时使用展示地址方法

### 4.3 已接入范围

当前已接入商城图片公共工具的管理端位置：

- 商品管理页
- 商品分类页
- 图片预览组件 `ImagePreview`

后续商城新页面涉及图片展示时，必须复用该工具，不再重复编写地址拼接逻辑。

## 5. 小程序处理规范

### 5.1 公共工具

小程序商城模块统一使用：

- `joolun-wx-ma/utils/util.js`

当前相关方法：

- `resolveResourceUrl(url, fallback)`
- `normalizePicUrlList(picUrls)`
- `getFirstPicUrl(picUrls, fallback)`

### 5.2 使用要求

小程序页面和组件禁止直接把接口返回的相对路径绑定到 `<image>`。

统一要求：

- 单图字段先走 `resolveResourceUrl`
- 多图字段先走 `normalizePicUrlList`
- 列表首图统一走 `getFirstPicUrl`

## 6. 后端处理规范

后端职责是保存和返回图片字段，不负责拼接运行环境域名。

后端需要保证：

- 单图字段按字符串返回
- 多图字段按数组语义返回
- 类型转换器兼容数组、JSON 字符串、逗号分隔字符串
- 不在业务代码中写死 `http://localhost`、`/dev-api`、生产域名等环境信息

## 7. 开发检查清单

新增或修改商城图片相关功能时，至少检查以下内容：

1. 数据库保存的是相对路径，不是完整域名
2. 接口返回的图片字段类型符合约定
3. 管理端展示使用了 `mall-image.js`
4. 小程序展示使用了 `utils/util.js` 中的图片方法
5. 列表页只取首图，详情页展示完整图集
6. 默认图、回退图逻辑明确，不出现空白图或错误相对路径

## 8. 推荐示例

### 8.1 管理端商品列表

```js
import { getMallFirstImageUrl } from "@/utils/mall-image";

const coverUrl = getMallFirstImageUrl(row.picUrls);
```

### 8.2 管理端分类列表

```js
import { resolveMallImageUrl } from "@/utils/mall-image";

const picUrl = resolveMallImageUrl(row.picUrl);
```

### 8.3 小程序商品详情

```js
const picUrls = util.normalizePicUrlList(goodsSpu.picUrls);
const coverUrl = util.getFirstPicUrl(goodsSpu.picUrls);
```
