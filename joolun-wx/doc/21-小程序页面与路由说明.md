# 小程序页面与路由说明

## 1. 页面结构

当前小程序页面全部在 `joolun-wx-ma/pages` 目录下，按业务分为以下几个分组：

- `home`
- `base`
- `goods`
- `shopping-cart`
- `order`
- `user`

## 2. 页面清单

当前在 `app.json` 中注册的页面如下：

- `pages/home/index`
- `pages/base/search/index`
- `pages/goods/goods-category/index`
- `pages/goods/goods-list/index`
- `pages/goods/goods-detail/index`
- `pages/shopping-cart/index`
- `pages/order/order-confirm/index`
- `pages/order/order-list/index`
- `pages/order/order-detail/index`
- `pages/order/after-sale-list/index`
- `pages/user/user-center/index`
- `pages/user/user-profile/index`
- `pages/user/user-address/list/index`
- `pages/user/user-address/form/index`

## 3. TabBar 说明

当前底部 `TabBar` 包含 4 个主入口：

- 首页：`pages/home/index`
- 分类：`pages/goods/goods-category/index`
- 购物车：`pages/shopping-cart/index`
- 我的：`pages/user/user-center/index`

当前没有单独新增“客服”作为底部 `TabBar` 项，客服入口采用页面内嵌方式接入。

## 4. 页面分组说明

### 4.1 首页组

页面：

- `pages/home/index`

当前职责：

- 首页轮播与商品推荐
- 新品、热销、商品列表展示
- 首页客服引导
- 订单和售后统计提示

### 4.2 搜索组

页面：

- `pages/base/search/index`

当前职责：

- 关键词搜索
- 商品搜索结果承接

### 4.3 商品组

页面：

- `pages/goods/goods-category/index`
- `pages/goods/goods-list/index`
- `pages/goods/goods-detail/index`

当前职责：

- 分类浏览商品
- 列表浏览商品
- 查看商品详情
- 选择商品规格
- 加入购物车
- 立即购买

当前商品详情页的规格选择使用底部弹窗方式，统一承接加购、立即购买和购物车改规格。

### 4.4 购物车组

页面：

- `pages/shopping-cart/index`

当前职责：

- 查看购物车
- 勾选结算商品
- 修改数量
- 修改规格
- 删除购物车商品
- 清理失效商品

### 4.5 订单组

页面：

- `pages/order/order-confirm/index`
- `pages/order/order-list/index`
- `pages/order/order-detail/index`
- `pages/order/after-sale-list/index`

当前职责：

- 确认订单
- 查看订单列表
- 查看订单详情
- 查看售后列表

当前售后列表按订单项维度展开展示。

### 4.6 用户组

页面：

- `pages/user/user-center/index`
- `pages/user/user-profile/index`
- `pages/user/user-address/list/index`
- `pages/user/user-address/form/index`

当前职责：

- 展示个人中心
- 展示会员身份
- 维护个人资料
- 绑定手机号
- 管理收货地址

## 5. 页面跳转关系

当前常见跳转关系如下：

- 首页可跳转搜索、分类、商品详情
- 分类页可跳转商品列表
- 商品列表可跳转商品详情
- 商品详情可跳转购物车和确认订单
- 购物车可跳转商品详情和确认订单
- 个人中心可跳转订单列表、售后列表、资料页、地址页
- 订单列表可跳转订单详情
- 订单详情可跳转售后列表或客服咨询

## 6. 登录态页面规则

### 6.1 统一规则

依赖登录态的页面，当前统一在 `app.initPage()` 完成后再请求接口。

标准写法如下：

```js
app.initPage()
  .then(() => {
    // 接口调用
  })
```

### 6.2 典型页面

当前以下页面明显依赖登录态：

- `pages/home/index`
- `pages/goods/goods-detail/index`
- `pages/shopping-cart/index`
- `pages/order/order-confirm/index`
- `pages/order/order-list/index`
- `pages/order/order-detail/index`
- `pages/order/after-sale-list/index`
- `pages/user/user-center/index`
- `pages/user/user-profile/index`
- `pages/user/user-address/list/index`
- `pages/user/user-address/form/index`

### 6.3 页面级复用

如果一个页面在 `onLoad` 和 `onShow` 都会刷新数据，建议优先做页面级 Promise 复用，避免重复触发登录初始化。

## 7. 页面参数约定

### 7.1 商品详情页

当前商品详情页主要接收以下参数：

- `id`
- `skuId`
- `shoppingCartId`
- `scene`

其中：

- `id` 表示商品主键
- `skuId` 用于带入默认选中的规格
- `shoppingCartId` 用于购物车改规格场景
- `scene` 用于扫码进入时还原商品主键

### 7.2 订单与售后页

订单和售后页面通常会携带以下参数：

- 订单状态
- 售后状态
- 订单主键

具体页面根据业务入口读取并转换为当前页筛选条件。

## 8. 客服入口说明

当前小程序客服采用微信原生客服能力，入口分布在以下页面：

- 首页
- 个人中心
- 订单详情相关区域

当前设计下，客服不单独占用底部 `TabBar`，而是跟随业务页面上下文进入。

## 9. 页面职责边界

### 9.1 首页

首页主要负责导流、推荐和服务引导，不负责深度交易编辑。

### 9.2 商品详情页

商品详情页主要负责规格选择和购买决策，不负责订单结果展示。

### 9.3 购物车页

购物车页主要负责结算前整理，不负责订单履约信息展示。

### 9.4 订单详情页

订单详情页主要负责履约和售后查看，不负责会员资料维护。

### 9.5 个人中心页

个人中心页主要负责身份展示和入口聚合，不负责商品浏览。

## 10. 二开约定

基于当前小程序页面结构进行二开时，建议遵循以下规则：

- 新页面优先归类到现有业务目录，不要把页面平铺在 `pages` 根下
- 依赖登录态的页面统一遵循 `app.initPage()` 先行规则
- 商品、购物车、订单、售后页面统一围绕 `sku_id` 和 `spec_info` 展示
- 会员身份展示统一读取 `mall_user`
- 客服入口优先放在业务上下文页面，不建议单独再做底部客服 `TabBar`
