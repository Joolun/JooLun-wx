# 商城接口分层说明

## 1. 文档范围

当前商城接口分为两条主线：

- 后台管理接口
- 小程序业务接口

两条接口线共用同一套商城领域模型，但面向的调用端、路径前缀和鉴权方式不同。

## 2. 分层结构

当前接口分层关系如下：

- 后台页面调用 `joolun-wx-ui/src/api/mall`
- 小程序页面调用 `joolun-wx-ma/utils/api.js`
- 后台接口控制器位于 `joolun-admin/src/main/java/com/joolun/web/controller/mall`
- 小程序接口控制器位于 `joolun-admin/src/main/java/com/joolun/web/api`
- 业务实现主要沉淀在 `joolun-mall`

## 3. 后台接口

### 3.1 控制器目录

- `GoodsCategoryController`

- `OrderInfoController`

- `OrderLogisticsController`

## 4. 小程序接口

### 4.1 控制器目录

- `joolun-admin/src/main/java/com/joolun/web/api`

当前控制器包括：

- `GoodsCategoryApi`
- `GoodsSpuApi`
- `MallConfigApi`
- `MallUserApi`
- `OrderInfoApi`
- `ShoppingCartApi`
- `UserAddressApi`

### 4.2 小程序前端封装

当前小程序统一通过 `joolun-wx-ma/utils/api.js` 封装接口请求。

该文件负责：

- 统一拼接请求地址
- 统一注入 `app-id`
- 统一注入 `third-session`
- 统一处理 `60001` session 过期

## 5. 鉴权口径

### 5.1 后台接口鉴权

后台接口当前主要使用：

- 登录态
- 权限点 `@PreAuthorize`

页面是否能看到按钮和接口是否允许调用，都会同时受权限点控制。

### 5.2 小程序接口鉴权

小程序接口当前主要依赖：

- 请求头 `app-id`
- 请求头 `third-session`

除登录接口和支付回调、退款回调等放行接口外，其余接口默认依赖 `third-session`。

## 6. 调用规则

### 6.1 后台页面调用规则

后台页面应优先通过 `src/api/mall` 调用接口，不直接在页面里手写请求地址。

### 6.2 小程序页面调用规则

小程序页面应优先通过 `app.api` 调用接口，并确保在 `app.initPage()` 之后调用依赖登录态的请求。

### 6.3 领域边界规则

基于当前结构进行二开时，建议遵循以下边界：

- 后台管理能力继续挂在 `/mall` 相关控制器下
- 小程序业务能力继续挂在 `/weixin/api/ma/` 路径下
- 后台和小程序共用同一业务口径，不要分别维护两套字段语义
- 页面层优先复用现有 API 封装，不要把请求分散写在多个位置

## 7. 常见映射关系

当前典型映射关系如下：

- 后台会员页 `malluser/index.vue` -> `src/api/mall/malluser.js` -> `MallUserController`
- 后台订单页 `orderinfo/index.vue` -> `src/api/mall/orderinfo.js` -> `OrderInfoController`
- 后台售后页 `aftersale/index.vue` -> `src/api/mall/aftersale.js` -> `OrderItemController`
- 小程序个人中心 `pages/user/user-center/index.js` -> `app.api.mallUserGet` / `app.api.orderCountAll`
- 小程序商品详情 `pages/goods/goods-detail/index.js` -> `app.api.goodsGet` / `app.api.shoppingCartAdd`
- 小程序订单页 `pages/order/order-detail/index.js` -> `app.api.orderGet` / `app.api.orderRefunds`
