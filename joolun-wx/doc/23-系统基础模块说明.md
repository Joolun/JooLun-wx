# 系统基础模块说明

## 1. 范围说明

本文档说明项目中的系统基础能力，覆盖以下模块：

- `joolun-admin`
- `joolun-framework`
- `joolun-common`
- `joolun-system`
- `joolun-quartz`
- `joolun-generator`

这些模块共同组成后台管理平台的登录认证、权限控制、系统管理、监控运维、代码生成和通用基础设施。

## 2. 模块结构

### 2.1 `joolun-admin`

统一后台启动模块，负责：

- Spring Boot 启动入口
- `application.yml` 配置装配
- 后台控制器暴露
- OpenAPI / Swagger 入口
- 系统、监控、工具、微信、商城等控制器聚合

### 2.2 `joolun-framework`

框架层模块，负责：

- Spring Security 安全配置
- JWT 鉴权过滤
- Redis 登录态续期
- MyBatis Plus、数据源、线程池等基础配置
- 全局 Web 相关扩展

主要目录包括：

- `aspectj`
- `config`
- `datasource`
- `interceptor`
- `security`
- `web`

### 2.3 `joolun-common`

公共模块，负责：

- 常量定义
- 基础领域对象
- 通用注解
- 异常体系
- Redis 封装
- 常用工具类
- XSS、防重复提交、数据脱敏等能力

主要目录包括：

- `annotation`
- `constant`
- `core`
- `exception`
- `utils`
- `sensitive`
- `xss`

### 2.4 `joolun-system`

系统业务模块，负责：

- 用户、角色、菜单、部门、岗位
- 字典、参数配置、公告
- 登录日志、操作日志、在线用户

主要目录包括：

- `domain`
- `mapper`
- `service`

### 2.5 `joolun-quartz`

定时任务模块，负责：

- 任务定义管理
- 任务执行日志
- 定时任务启停和立即执行

### 2.6 `joolun-generator`

代码生成模块，负责：

- 读取表结构
- 生成后端代码模板
- 生成前端页面模板
- 代码预览与下载

## 4. 登录与认证

### 4.1 登录入口

后台登录主要通过以下接口完成：

- `POST /login`
- `GET /getInfo`
- `GET /getRouters`

对应控制器位于：

- `com.joolun.web.controller.system.SysLoginController`

### 4.2 认证方式

当前后台使用：

- Spring Security
- JWT 令牌
- Redis 登录态缓存

登录成功后，服务端会返回 token，前端后续请求通过 `Authorization` 请求头携带。

### 4.3 登录态管理

`TokenService` 当前负责：

- 生成 JWT
- 从请求头解析 token
- 通过 Redis 读取 `LoginUser`
- 自动续期
- 记录登录设备、浏览器、IP 和登录地

默认配置项位于：

- `token.header`
- `token.secret`
- `token.expireTime`

### 4.4 匿名放行地址

`SecurityConfig` 当前默认放行以下核心路径：

- `/login`
- `/register`
- `/captchaImage`
- `/weixin/portal/**`
- `/weixin/api/**`
- `/joolun-wiki/**`
- Swagger 和静态资源相关路径

除放行地址外，其余接口默认都需要鉴权。

## 5. 权限模型

### 5.1 菜单与权限点

后台权限以 `sys_menu` 为核心，菜单既承担：

- 路由菜单
- 按钮权限
- 接口权限标识

控制器中常见写法：

- `@PreAuthorize("@ss.hasPermi('system:user:list')")`
- `@PreAuthorize("@ss.hasPermi('wxmp:wxuser:index')")`

### 5.2 角色授权

角色负责聚合菜单权限和部门数据范围，典型关系如下：

- 用户绑定多个角色
- 角色绑定多个菜单
- 角色可绑定多个部门

### 5.3 路由生成

后台登录后，前端菜单树通过 `GET /getRouters` 获取，后端会根据当前用户权限动态构造路由数据。

## 10. 二开使用规范

### 10.1 新增后台模块

新增后台模块时，建议保持以下结构一致：

- 后端增加 `Controller + Service + Mapper + Domain`
- 前端增加 `src/api` 接口文件
- 前端增加 `src/views` 页面目录
- 菜单和权限点同步配置到 `sys_menu`

### 10.2 新增权限点

建议每个页面至少拆分以下权限：

- 列表查询
- 详情查看
- 新增
- 编辑
- 删除

### 10.3 新增字典

通用状态、类型、分类字段优先使用数据字典维护，避免把展示文案散落在多个页面中。

### 10.4 新增操作日志

后台关键写操作建议保留操作日志，便于审计和问题排查。
