# 商城后台页面使用说明

## 1. 页面范围

当前商城后台页面主要位于 `joolun-wx-ui/src/views/mall` 目录下，核心页面包括：

- `goodscategory`
- `goodsspu`
- `malluser`
- `orderinfo`
- `aftersale`

这些页面共同覆盖商品、会员、订单、售后的基础运营工作台。

## 2. 页面入口

### 2.1 商品分类

路径：

- `src/views/mall/goodscategory/index.vue`

当前职责：

- 维护商城分类树
- 新增、编辑、删除分类
- 调整分类启用状态
- 维护分类图片和父子级关系

### 2.2 商品管理

路径：

- `src/views/mall/goodsspu/index.vue`

当前职责：

- 维护商品基础资料
- 维护主图、描述、卖点、分类
- 维护单规格或多规格模式
- 维护规格项、规格值和 `SKU` 组合
- 批量上架
- 批量下架

当前页面是商城商品运营的核心入口。

### 2.3 商城用户

路径：

- `src/views/mall/malluser/index.vue`
- `src/views/mall/malluser/detail.vue`

当前职责：

- 会员列表查询
- 会员画像摘要
- 标签分布查看
- 标签库管理
- 批量启用、禁用、备注、打标签
- 查看会员详情
- 新增会员跟进记录

### 2.4 订单管理

路径：

- `src/views/mall/orderinfo/index.vue`

当前职责：

- 按订单状态查询订单
- 查看订单摘要统计
- 按会员维度筛选订单
- 查看订单详情
- 发货
- 取消未支付订单
- 处理退款审核

### 2.5 售后工作台

路径：

- `src/views/mall/aftersale/index.vue`

当前职责：

- 按订单项维度查看售后
- 按订单号、会员、商品筛选售后
- 按售后状态快捷筛选
- 查看售后摘要统计
- 审核退款

## 3. 路由说明

### 3.1 常规菜单页

商城后台常规页面由后台菜单控制显示，实际组件主要对应以下路径：

- `/mall/goodscategory`
- `/mall/goodsspu`
- `/mall/malluser`
- `/mall/orderinfo`
- `/mall/aftersale`

### 3.2 隐藏详情路由

当前路由中额外声明了一个隐藏详情页：

- `/mall/malluser-detail/index`

对应组件：

- `src/views/mall/malluser/detail.vue`

这个页面用于从会员列表、订单页等位置跳转查看会员详情，不作为左侧菜单独立展示。

## 4. 页面分工

### 4.1 商品分类页

适合处理以下工作：

- 梳理商城分类结构
- 控制分类是否可用
- 维护分类封面图

不建议在这个页面处理：

- 商品上下架
- 规格组合
- 库存价格

### 4.2 商品管理页

适合处理以下工作：

- 新增商品
- 编辑商品资料
- 维护多规格 `SKU`
- 调整库存与价格
- 批量上下架

这是商品运营与商品编辑的主页面。

### 4.3 商城用户页

适合处理以下工作：

- 查看商城用户和会员分布
- 进行会员筛选
- 做标签运营
- 做客服跟进和备注

当前会员定义已内置在该页面口径中，绑定手机号的用户会被视为会员。

### 4.4 订单管理页

适合处理以下工作：

- 查看订单列表
- 处理发货
- 查看订单明细
- 进行退款审核入口跳转
- 从订单维度回看会员信息

### 4.5 售后工作台

适合处理以下工作：

- 集中处理退款申请
- 审核退款通过或拒绝
- 查看售后时间线
- 快速定位某个订单项的售后状态

## 5. 权限口径

当前页面能力按权限点控制，常见权限前缀包括：

- `mall:goodscategory:*`
- `mall:goodsspu:*`
- `mall:malluser:*`
- `mall:orderinfo:*`
- `mall:aftersale:*`

页面内的新增、编辑、删除、查看、导出、审核、批量操作，都会根据权限点决定是否展示入口。

## 6. 页面联动关系

当前后台页面存在以下联动：

- 商品管理依赖商品分类
- 会员详情可从会员页直接进入
- 订单页可跳转会员详情
- 订单页和售后页都会关联会员信息
- 售后处理结果会回写订单与会员两侧日志

## 7. 使用建议

### 7.1 商品运营顺序

建议的后台操作顺序为：

1. 先维护分类
2. 再创建商品
3. 再配置规格与 `SKU`
4. 最后上架商品

### 7.2 会员运营顺序

建议的后台操作顺序为：

1. 先筛选目标会员
2. 再查看标签分布和会员摘要
3. 再批量打标签或备注
4. 需要深入跟进时进入会员详情页

### 7.3 订单与售后处理顺序

建议的后台操作顺序为：

1. 在订单页确认订单状态与支付状态
2. 发货和取消在订单页处理
3. 退款审核在售后工作台集中处理
4. 需要回看服务记录时结合订单日志和会员日志查看

## 8. 二开约定

基于当前后台页面进行二开时，建议遵循以下规则：

- 商品编辑能力优先继续集中在 `goodsspu` 页面
- 会员运营能力优先集中在 `malluser` 页面
- 售后审核能力优先集中在 `aftersale` 页面
- 会员详情类页面优先使用隐藏路由方式接入，不新增重复菜单
- 页面口径要与当前数据库设计和接口字段保持一致
