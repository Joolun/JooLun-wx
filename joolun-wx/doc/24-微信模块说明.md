# 微信模块说明

## 1. 范围说明

本文档说明项目中的微信相关能力，覆盖以下部分：

- `joolun-weixin`
- `joolun-admin` 中的微信控制器
- `joolun-wx-ui` 中的公众号与小程序管理页面
- `joolun-admin` 中对小程序开放的 `/weixin/api/ma/*` 接口

当前微信模块包含两类能力：

- 公众号能力
- 小程序能力

## 2. 模块结构

### 2.1 后端模块目录

`joolun-weixin` 当前主要目录包括：

- `builder`
- `config`
- `constant`
- `entity`
- `handler`
- `interceptor`
- `mapper`
- `service`
- `utils`

### 2.2 各目录职责

`config` 负责：

- 公众号配置装配
- 小程序配置装配
- 微信支付配置装配

`entity` 负责：

- 微信用户
- 微信消息
- 自定义菜单
- 自动回复
- 小程序登录 DTO
- 第三方会话对象

`handler` 负责：

- 公众号消息路由处理
- 关注、取关、扫码、菜单点击、地理位置等事件处理

`service` 负责：

- 微信用户管理
- 自动回复管理
- 菜单管理
- 消息管理

`utils` 负责：

- 微信文件处理
- JSON 处理
- 小程序工具类
- `third-session` 上下文持有

## 3. 核心数据表

微信模块当前主要使用以下 `wx_` 表：

- `wx_user`
- `wx_msg`
- `wx_menu`
- `wx_auto_reply`

### 3.1 `wx_user`

微信用户主表，统一沉淀公众号用户和小程序用户。

关键字段包括：

- `app_type`：应用类型，`1` 小程序，`2` 公众号
- `subscribe`：关注状态
- `open_id`
- `union_id`
- `nick_name`
- `phone`
- `tagid_list`
- `session_key`

### 3.2 `wx_msg`

微信消息表，用于保存：

- 用户发给公众号的消息
- 公众号发给用户的消息
- 事件消息
- 客服消息记录

### 3.3 `wx_menu`

公众号自定义菜单表，用于保存：

- 菜单层级
- 菜单类型
- 跳转链接
- 小程序跳转参数
- 菜单点击后的回复配置

### 3.4 `wx_auto_reply`

自动回复表，用于保存：

- 关注时回复
- 消息回复
- 关键词回复

## 4. 公众号能力

### 4.1 接入入口

公众号服务端接入入口为：

- `GET /weixin/portal/{appid}`
- `POST /weixin/portal/{appid}`

对应控制器：

- `WxPortalController`

作用包括：

- 微信服务器接入校验
- 接收公众号消息
- 校验签名
- 交给消息路由器处理

### 4.2 消息路由

`WxMpConfiguration` 当前注册了公众号消息路由，默认覆盖以下场景：

- 客服会话事件
- 门店审核事件
- 自定义菜单点击事件
- 菜单跳转事件
- 关注事件
- 取关事件
- 地理位置事件
- 扫码事件
- 普通消息兜底处理

对应处理器包括：

- `LogHandler`
- `KfSessionHandler`
- `StoreCheckNotifyHandler`
- `MenuHandler`
- `SubscribeHandler`
- `UnsubscribeHandler`
- `LocationHandler`
- `ScanHandler`
- `MsgHandler`
- `NullHandler`

### 4.3 后台页面

公众号后台页面目录位于：

- `joolun-wx-ui/src/views/wxmp/wxautoreply`
- `joolun-wx-ui/src/views/wxmp/wxdraft`
- `joolun-wx-ui/src/views/wxmp/wxfreepublish`
- `joolun-wx-ui/src/views/wxmp/wxmaterial`
- `joolun-wx-ui/src/views/wxmp/wxmenu`

### 4.4 后台控制器

公众号后台控制器包括：

- `WxAutoReplyController`

### 4.5 主要使用场景

公众号模块当前适合处理以下场景：

## 5. 小程序能力

### 5.1 小程序配置

小程序配置由 `WxMaConfiguration` 装配，支持多 `appId` 配置。

当前单项配置包含：

- `appId`
- `secret`
- `token`
- `aesKey`
- `msgDataFormat`
- `mchId`
- `mchKey`
- `keyPath`

### 5.2 小程序会话

当前小程序登录态核心对象为 `ThirdSession`，包含：

- `wxUserId`
- `appId`
- `sessionKey`
- `openId`

小程序侧依赖 `third-session` 建立服务端登录态。

### 5.3 小程序开放接口

当前小程序对外接口统一以 `/weixin/api/ma/*` 暴露，主要包括：

- `/weixin/api/ma/goodscategory`
- `/weixin/api/ma/goodsspu`
- `/weixin/api/ma/mall/config`
- `/weixin/api/ma/malluser`
- `/weixin/api/ma/wxuser`
- `/weixin/api/ma/orderinfo`
- `/weixin/api/ma/shoppingcart`
- `/weixin/api/ma/useraddress`

这些接口位于：

- `joolun-admin/src/main/java/com/joolun/web/api`

### 5.4 小程序管理页面

后台当前提供的小程序微信用户页面位于：

- `joolun-wx-ui/src/views/wxma/wxuser`

对应接口文件位于：

- `joolun-wx-ui/src/api/wxma/wxuser.js`

### 5.5 登录与访问规范

小程序接口调用规范详见：

- `20-小程序登录与接口访问规范.md`
- `21-小程序页面与路由说明.md`

当前约定是页面在调用需要登录态的接口前，先执行：

```js
app.initPage()
  .then(() => {
    // 接口调用
  })
```

## 6. 微信用户口径

### 6.1 统一用户主表

微信模块当前使用 `wx_user` 统一沉淀：

- 小程序用户
- 公众号用户

其中：

- `app_type = 1` 表示小程序用户
- `app_type = 2` 表示公众号用户

### 6.2 关注状态

`wx_user.subscribe` 口径如下：

- `1`：已关注
- `0`：未关注
- `2`：网页授权用户

### 6.3 标签字段

`wx_user.tagid_list` 用于保存微信侧标签列表，后台公众号用户页支持按标签筛选和批量打标签。

## 9. 二开使用规范

### 9.1 新增公众号事件

新增公众号事件处理时，建议按以下方式扩展：

- 先新增 `handler`
- 再在 `WxMpConfiguration` 中注册路由规则
- 如需留痕，再补消息表或业务日志

### 9.2 新增微信接口

新增微信相关接口时，建议明确区分：

- 后台管理接口
- 公众号服务端回调接口
- 小程序开放接口

路径建议保持现有风格：

- 后台管理接口直接暴露控制器路径
- 小程序接口统一放到 `/weixin/api/ma/*`

### 9.3 用户体系使用建议

如果是商城业务身份，优先使用 `mall_user`；
如果是微信渠道身份，优先使用 `wx_user`；
涉及登录授权、OpenId、UnionId、标签、关注状态等微信原生能力时，统一以 `wx_user` 为准。
