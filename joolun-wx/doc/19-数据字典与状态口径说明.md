# 数据字典与状态口径说明

## 1. 说明

本文档用于统一商城模块的字段口径、状态值含义、页面展示文案和日志类型说明，便于后台、后端、小程序在开发和联调时保持一致。

## 2. 通用字段口径

### 2.1 布尔值口径

商城模块通用布尔值统一使用字符串：

- `0`：否
- `1`：是

对应常量定义：

- `CommonConstants.NO = "0"`
- `CommonConstants.YES = "1"`

### 2.2 主键与关联字段

- 主键统一使用 `id`
- 商品主档主键使用 `spu_id`
- 商品规格明细主键使用 `sku_id`
- 商城用户主键使用 `mall_user.id`
- 订单主表关联用户使用 `user_id`
- 订单明细关联订单使用 `order_id`
- 订单明细关联商品主档使用 `spu_id`
- 订单明细关联商品规格使用 `sku_id`

### 2.3 时间字段口径

商城模块常见时间字段说明如下：

- `create_time`：创建时间
- `update_time`：更新时间
- `payment_time`：支付时间
- `delivery_time`：发货时间
- `receiver_time`：收货时间
- `closing_time`：订单关闭或取消时间
- `refund_apply_time`：售后申请时间
- `refund_audit_time`：售后审核时间
- `refund_success_time`：退款完成时间
- `register_time`：商城用户注册时间
- `member_time`：会员入会时间
- `last_login_time`：最后登录时间

## 3. 商品相关口径

### 3.1 商品上下架

商品主档 `mall_goods_spu.status` 用于表示商品当前可售状态，后台和小程序展示时应统一按“上架 / 下架”理解。

### 3.2 SKU 启用状态

商品规格明细 `mall_goods_sku.status` 用于表示当前 SKU 是否参与销售，页面展示时应统一按“启用 / 停用”理解。

### 3.3 规格信息

- `spec_info`：用于展示规格组合文案，例如 `颜色：黑色；尺码：L`
- `mall_goods_spu_spec`：用于记录商品可选规格维度
- `mall_goods_sku_spec_value`：用于记录某个 SKU 对应的规格值组合

购物车、订单明细、小程序商品详情均以 `sku_id + spec_info` 作为展示和下单依据。

## 4. 商城用户与会员口径

### 4.1 商城用户状态

`mall_user.status` 含义如下：

- `0`：正常
- `1`：禁用

### 4.2 会员身份

当前系统将“已绑定手机号的商城用户”定义为会员，字段口径如下：

- `member_flag = 0`：未入会
- `member_flag = 1`：已入会

辅助字段说明：

- `member_time`：入会时间
- `mobile`：手机号，当前会员判定的关键字段
- `user_no`：商城会员编号

### 4.3 注册来源

后台页面中的注册来源文案统一如下：

- `1`：小程序
- `2`：公众号
- `3`：H5
- `4`：APP
- 其他值：其他

### 4.4 活跃场景筛选

商城用户列表中的活跃场景筛选值如下：

- `active7`：近 7 天活跃
- `wake30`：近 30 天可唤醒
- `silent30`：30 天沉睡
- `neverLogin`：从未登录

### 4.5 会员页面分层口径

后台会员详情页当前使用以下分层展示：

会员分层标签：

- `已禁用`
- `待转化`
- `复购会员`
- `成交会员`
- `下单待支付`
- `新入会用户`

活跃分层标签：

- `高活跃`
- `可唤醒`
- `沉睡中`
- `暂无记录`

价值分层标签：

- `高价值`
- `成长中`
- `已成交`
- `待培育`

### 4.6 运营标签

后台会员页当前内置的标签示例包括：

- `高价值会员`
- `复购会员`
- `新会员`
- `待转化`
- `沉睡召回`
- `短信唤醒`
- `活动邀约`
- `4月首购`

## 5. 订单口径

### 5.1 主订单状态

`mall_order_info.status` 统一含义如下：

- `0`：待付款
- `1`：待发货
- `2`：待收货
- `3`：已完成
- `5`：已取消

### 5.2 支付状态

`mall_order_info.is_pay` 统一含义如下：

- `0`：未支付
- `1`：已支付

### 5.3 订单金额字段

- `sales_price`：商品总金额
- `freight_price`：运费金额
- `payment_price`：实付金额

页面展示金额时统一保留两位小数。

### 5.4 订单维度统计口径

后台和会员详情页中常用统计字段说明如下：

- `order_count`：累计下单次数
- `consume_count`：累计支付订单数
- `refund_count`：累计退款单数
- `consume_amount`：累计消费金额
- `wait_pay_order_count`：待付款订单数
- `wait_delivery_order_count`：待发货订单数
- `wait_receive_order_count`：待收货订单数

## 6. 售后口径

### 6.1 售后识别方式

当前售后以订单项 `mall_order_item` 为处理单位，不以整单为单位处理。

### 6.2 售后状态

售后状态由 `status` 和 `is_refund` 组合表达：

- `status = 1`：退款申请中
- `status = 2`：退款被拒绝
- `status = 3`：退款待回调
- `is_refund = 1`：已退款

页面层统一展示文案如下：

- `退款申请中`
- `退款已拒绝` 或 `退款被拒绝`
- `等待退款回调` 或 `退款待回调`
- `已退款`
- `无售后`

### 6.3 售后关键字段

- `refund_reason`：售后申请原因
- `refund_apply_time`：申请时间
- `refund_audit_remark`：审核备注
- `refund_audit_time`：审核时间
- `refund_success_time`：退款完成时间

## 8. 日志类型口径

### 8.1 会员运营日志

`mall_user_operate_log.operate_type` 当前常见取值如下：

- `FOLLOW_UP`：跟进记录
- `REFUND_APPROVE`：退款通过
- `REFUND_REJECT`：退款拒绝
- `REFUND_SUCCESS`：退款到账
- `TAG`：标签变更
- `REMARK`：备注变更
- `STATUS`：状态变更
- `PROFILE`：资料修改

### 8.2 订单操作日志

`mall_order_operate_log.operate_type` 当前常见取值如下：

- `CREATE`：创建订单
- `PAY_SUCCESS`：支付成功
- `DELIVERY`：后台发货
- `RECEIVE`：用户收货
- `USER_CANCEL`：用户取消
- `ADMIN_CANCEL`：后台取消
- `REFUND_APPLY`：发起退款申请
- `REFUND_REJECT`：退款拒绝
- `REFUND_APPROVE`：退款通过
- `REFUND_SUCCESS`：退款完成

## 9. 页面展示统一要求

### 9.1 状态展示要求

- 后端枚举、后台文案、小程序文案需保持同一口径
- 已退款优先级高于售后处理中状态
- 售后文案统一围绕订单项展示，不按整单混合计算
- 会员身份统一按手机号绑定结果判定

### 9.2 展示字段优先级

- 商品规格优先显示 `spec_info`
- 订单金额优先显示 `payment_price`
- 售后说明优先显示 `refund_reason` 与 `refund_audit_remark`
- 会员展示优先显示 `nick_name`，为空时可回退 `real_name`
