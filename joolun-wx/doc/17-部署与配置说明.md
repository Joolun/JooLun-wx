# 部署与配置说明

## 1. 项目组成

当前工作区包含 3 个主要工程：

- `joolun-wx`：Java 后端
- `joolun-wx-ui`：后台前端
- `joolun-wx-ma`：微信小程序

部署时通常需要分别处理：

- 数据库
- 后端服务
- 后台前端静态资源
- 小程序请求地址与微信配置

## 2. 后端环境要求

当前后端核心环境如下：

- Java 17
- Maven 3+
- MySQL 8
- Redis

Maven 聚合工程位于：

- `joolun-wx/pom.xml`

当前聚合模块包括：

- `joolun-admin`
- `joolun-framework`
- `joolun-system`
- `joolun-common`
- `joolun-quartz`
- `joolun-generator`
- `joolun-weixin`
- `joolun-mall`

## 3. 后端配置入口

### 3.1 主配置文件

当前后端主配置文件位于：

- `joolun-wx/joolun-admin/src/main/resources/application.yml`
- `joolun-wx/joolun-admin/src/main/resources/application-druid.yml`

### 3.2 端口与上传目录

当前主配置中重点关注：

- `server.port`
- `joolun.profile`

其中：

- `server.port` 决定后端服务监听端口
- `joolun.profile` 决定上传文件落盘目录

### 3.3 数据库配置

当前数据库连接位于 `application-druid.yml`，重点字段包括：

- `spring.datasource.druid.master.url`
- `spring.datasource.druid.master.username`
- `spring.datasource.druid.master.password`

### 3.4 Redis 配置

当前 Redis 配置位于 `application.yml`，重点字段包括：

- `spring.data.redis.host`
- `spring.data.redis.port`
- `spring.data.redis.database`
- `spring.data.redis.password`

### 3.5 Token 配置

当前登录态配置位于 `application.yml`，重点字段包括：

- `token.header`
- `token.secret`
- `token.expireTime`

### 3.6 微信配置

当前微信相关配置位于 `application.yml`，重点分组包括：

- `wx.mp.configs`
- `wx.ma.configs`

小程序支付与登录常用字段包括：

- `appId`
- `secret`
- `mchId`
- `mchKey`
- `keyPath`

### 3.7 商城回调域名

当前商城支付和退款回调依赖：

- `mall.notify-host`

这个地址需要能从外网访问到后端服务。

### 4.1 基础脚本

当前数据库脚本目录位于：

- `joolun-wx/sql`

### 4.2 商城相关脚本

- `joolun_ry.sql`
- `01mall_multi_sku.sql`
- `00mall_table_prefix_upgrade.sql`
- `mall_user_upgrade.sql`
- `mall_user_member_upgrade.sql`

## 5. 后端启动

### 5.1 本地启动

本地启动前应先准备：

1. MySQL
2. Redis
3. 正确的 `application.yml`
4. 正确的 `application-druid.yml`

### 5.2 打包

当前后端使用 Maven 打包，通常从聚合工程根目录执行。

### 5.3 运行对象

当前可运行的后端入口是 `joolun-admin` 模块，对外统一提供后台接口和小程序接口。

## 6. 后台前端配置

### 6.1 项目入口

后台前端项目位于：

- `joolun-wx-ui`

### 6.2 技术栈

当前后台前端使用：

- Vue 3
- Vite
- Element Plus
- Avue

### 6.3 常用脚本

当前 `package.json` 中常用脚本包括：

- `dev`
- `build:prod`
- `build:stage`
- `preview`

### 6.4 部署方式

后台前端打包后输出静态资源，通常部署到：

- Nginx
- 静态资源服务器
- 后端同域静态站点

## 7. 小程序配置

### 7.1 配置文件

小程序主要环境配置位于：

- `joolun-wx-ma/config/env.js`

### 7.2 重点字段

当前小程序重点配置包括：

- `basePath`
- `adEnable`
- `adBannerID`
- `adInsertScreenID`
- `adIncentiveID`

其中：

- `basePath` 是小程序请求后端的基础地址
- 广告相关字段按实际业务需要启用或关闭

### 7.3 联调规则

本地联调时：

- `basePath` 不能只写 `localhost`
- 真机调试时应使用局域网可访问地址
- 手机和开发机应在同一网络

## 8. 小程序发布

### 8.1 页面配置

小程序页面路由位于：

- `joolun-wx-ma/app.json`

### 8.2 登录依赖

小程序正式运行依赖：

- 微信小程序 `appId`
- 后端微信配置
- 可访问的后端域名
- 正常可用的 Redis session 存储

### 8.3 包体控制

当前小程序已清理部分大图资源，后续发布时仍应持续关注主包大小，避免超过微信包体限制。

## 9. 反向代理与域名

当前部署时建议明确以下地址关系：

- 后台前端访问域名
- 后端 API 域名
- 小程序请求域名
- 微信支付回调域名

如果通过网关或 Nginx 代理，应保证：

- 后台接口可正常转发
- 小程序接口 `/weixin/api/ma/**` 可正常转发
- 回调地址能直达后端

## 10. 部署检查清单

当前建议上线前至少检查以下内容：

1. 数据库表结构和当前代码一致
2. Redis 连接正常
3. 后端端口和域名可访问
4. `mall.notify-host` 可公网访问
5. 小程序 `basePath` 指向正确环境
6. 微信 `appId`、`secret`、支付商户配置正确
7. 上传目录具备写权限
8. 后台静态资源打包版本与后端接口版本一致

## 11. 二开约定

基于当前项目进行部署与配置二开时，建议遵循以下规则：

- 配置项优先继续集中在现有配置文件，不要分散到多处
- 小程序环境地址优先通过 `env.js` 维护
- 微信、支付、回调域名统一从后端配置读取
- 部署文档应围绕当前 3 个工程分别维护
