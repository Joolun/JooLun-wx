# 商城用户与会员设计

## 1. 当前定义

当前系统中，“商城用户”和“会员”属于同一条用户主线：

- `mall_user` 表示商城用户
- 绑定手机号后的商城用户，视为会员

也就是说，当前会员并不是单独一张主表，而是 `mall_user` 在满足入会条件后的身份状态。

## 2. 用户模型

### 2.1 `mall_user` 的职责

`mall_user` 是商城领域的用户主数据表，负责承接商城自己的身份、资料和经营统计。

它和微信渠道用户的关系如下：

- `wx_user` 负责微信侧渠道身份
- `mall_user` 负责商城业务身份
- `mall_user.wx_user_id` 关联对应的微信用户

这种结构下，商城的订单、购物车、收货地址、会员运营等能力，都以 `mall_user` 为主线组织。

### 2.2 主要字段分组

#### 身份标识

- `id`
- `user_no`
- `wx_user_id`
- `open_id`
- `union_id`

- `nick_name`

#### 会员身份

#### 运营资料

- `user_tag`

#### 用户行为统计

## 3. 会员定义

### 3.1 入会规则

当前会员规则非常明确：

- 只要商城用户成功绑定手机号，即视为会员
- `member_flag = 1`
- `member_time` 记录首次成为会员的时间

### 3.2 非会员状态

满足以下情况时，当前视为非会员：

- 尚未绑定手机号
- `member_flag = 0`

### 3.3 会员状态和账号状态的区别

会员身份和账号可用状态是两套字段：

- `member_flag` 表示是否为会员
- `status` 表示账号是否启用

当前口径下：

- 会员可以被后台禁用
- 非会员也可以保留商城用户档案

## 4. 标签与运营留痕

### 4.1 标签库

当前系统使用 `mall_user_tag` 维护会员标签库。

主要字段包括：

- `tag_name`
- `status`
- `sort`
- `remark`

后台可基于标签库做以下操作：

- 查看标签库列表
- 启停标签
- 设置排序
- 查看标签使用人数
- 给会员打标签

### 4.2 会员运营日志

当前系统使用 `mall_user_operate_log` 记录会员维度的运营留痕。

主要字段包括：

- `user_id`
- `operate_type`
- `operate_title`
- `operate_content`
- `operator_id`
- `operator_name`
- `extra_info`
- `create_time`

当前常见日志类型包括：

- `PROFILE`
- `STATUS`
- `TAG`
- `REMARK`
- `FOLLOW_UP`
- `DELIVERY`
- `ADMIN_CANCEL`
- `REFUND_APPROVE`
- `REFUND_REJECT`

这张表既承接会员资料变更，也承接客服与订单服务动作在会员维度的同步留痕。

## 5. 后台能力

### 5.1 会员列表

后台商城用户页当前支持以下能力：

- 分页查询
- 导出当前筛选结果
- 按会员编号、昵称、姓名、手机号查询
- 按会员身份、状态、注册来源筛选
- 按注册时间、最近登录时间、入会时间筛选
- 按累计消费、累计下单次数筛选
- 按活跃场景筛选

### 5.2 会员摘要

后台会员页顶部当前提供会员摘要和当前页洞察能力，便于运营查看：

- 用户总量
- 会员量
- 支付用户量
- 累计消费
- 标签分布
- 当前筛选结果下的转化洞察

### 5.3 批量运营

后台当前支持以下批量操作：

- 批量启用
- 批量禁用
- 批量备注
- 批量打标签

### 5.4 会员详情

会员详情页当前可查看：

- 基础资料
- 会员身份
- 地区信息
- 注册来源
- 最近登录
- 最近下单
- 最近支付
- 收货地址数量
- 待付款、待发货、待收货订单数
- 购物车数量
- 运营日志

### 5.5 跟进记录

后台支持按会员新增跟进记录，用于沉淀客服、运营或售后处理说明。

## 6. 小程序链路

### 6.1 页面入口

当前与商城用户、会员直接相关的小程序页面包括：

- `pages/user/user-center`
- `pages/user/user-profile`
- `pages/user/user-address/list`
- `pages/user/user-address/form`

### 6.2 登录后的用户初始化

依赖登录态的会员接口，当前统一在 `app.initPage()` 完成后再发起。

常见写法如下：

```js
app.initPage()
  .then(() => {
    return app.api.mallUserGet()
  })
```

### 6.3 资料同步

当前小程序资料同步链路包括：

- 通过 `mallUserGet` 拉取商城用户档案
- 通过 `mallUserSave` 同步微信头像和昵称
- 通过 `mallUserUpdate` 维护真实姓名、性别、生日

### 6.4 会员入会

当前小程序通过绑定手机号完成会员身份建立。

绑定成功后，页面会同步刷新：

- `mobile`
- `member_flag`
- `member_time`

### 6.5 个人中心展示

当前小程序个人中心会展示以下会员信息：

- 会员身份文案
- 累计下单次数
- 累计支付次数
- 累计退款次数
- 累计消费金额
- 售后数量
- 售后处理中数量

## 8. 使用规范

基于当前会员设计进行二开时，建议遵循以下规则：

- 商城业务表中的 `user_id` 统一引用 `mall_user.id`
- 不要把“微信用户”和“商城用户”当成同一概念混用
- 会员身份统一以 `member_flag` 和 `member_time` 作为展示与筛选依据
- 标签筛选、客服备注、跟进记录优先写入商城域表，不要写回渠道用户表
- 会员页、小程序个人中心、订单详情等需要展示用户身份的地方，统一读取 `mall_user`
