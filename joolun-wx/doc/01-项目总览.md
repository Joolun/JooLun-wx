# 项目总览

## 1. 项目定位

本项目是基于 JooLun 微信生态能力建设的商城系统，当前以“小程序商城 + 商城后台管理 + 商城后端服务”为核心。

当前系统具备以下核心能力：

- 商品已支持多规格 SKU
- 商城表已统一使用 `mall_` 前缀
- 商城用户与微信渠道用户已做领域拆分
- 会员规则已明确为“绑定手机号后视为会员”
- 订单售后链路已补强
- 小程序客服采用微信原生客服方案

当前系统以商品、会员、订单、售后、客服等基础经营链路为主。

## 2. 代码仓结构

当前工作区包含 3 个主要工程：

### 2.1 后端工程

路径：`joolun-wx`

主要职责：

- 提供商城、微信、小程序相关后端接口
- 管理商城核心业务实体、Mapper、Service、Controller
- 管理数据库升级脚本

重点模块说明：

- `joolun-admin`
  统一后台入口，包含商城后台接口控制器
- `joolun-mall`
  商城核心业务模块，包含商品、订单、商城用户、购物车、收货地址等实体与服务
- `joolun-weixin`
  微信渠道能力模块，包含微信用户、小程序接入等能力
- `sql`
  数据初始化脚本与升级脚本目录
- `doc`
  项目文档目录

### 2.2 后台前端工程

路径：`joolun-wx-ui`

主要职责：

- 商城后台管理页面
- 商品维护、多规格 SKU 维护
- 商城用户管理
- 订单与售后工作台

当前商城相关页面主要位于：

- `src/views/mall/goodsspu`
- `src/views/mall/malluser`
- `src/views/mall/orderinfo`
- `src/views/mall/aftersale`
- `src/views/mall/goodscategory`

### 2.3 小程序工程

路径：`joolun-wx-ma`

主要职责：

- 小程序首页、分类、购物车、我的
- 商品详情与规格选择
- 下单、订单、售后
- 会员资料与手机号绑定
- 微信原生客服接入

当前主要页面目录：

- `pages/home`
- `pages/goods`
- `pages/shopping-cart`
- `pages/order`
- `pages/user`

## 4. 当前核心业务模型

### 4.1 商品模型

当前商品采用 `SPU + SKU` 双层模型：

- `mall_goods_spu`
  商品主档，承载展示层信息和价格区间
- `mall_goods_sku`
  真实销售单元，承载价格、库存、启用状态
- `mall_goods_spu_spec`
  当前 SPU 绑定的规格维度
- `mall_goods_sku_spec_value`
  当前 SKU 绑定的规格值组合

补充说明：

- `mall_shopping_cart` 已新增 `sku_id`、`spec_info`
- `mall_order_item` 已新增 `sku_id`、`spec_info`
- 小程序和后台均已围绕 SKU 进行下单、购物车和规格切换

### 4.2 用户与会员模型

当前系统区分两类用户：

- `wx_user`
  微信渠道用户
- `mall_user`
  商城领域用户

当前会员判定规则：

- 商城用户绑定手机号后，视为会员
- `mall_user.member_flag = 1` 表示已入会
- `mall_user.member_time` 表示入会时间

### 4.3 订单与售后模型

当前订单与售后相关核心表：

- `mall_order_info`
  订单主表
- `mall_order_item`
  订单明细快照表
- `mall_order_operate_log`
  订单操作留痕
- `mall_user_operate_log`
  会员运营与客服跟进行为留痕

售后当前采用“基于订单项”的处理思路：

- 一个订单可对应多个订单项
- 每个订单项可以进入自己的退款/售后状态流转
- 小程序和后台均已支持从订单维度联动查看售后状态

## 5. 当前能力清单

### 5.1 商品能力

- 完成单规格商品改多规格 SKU
- 后台商品页支持规格维度、规格值、SKU 组合维护
- 小程序商品详情已改为底部规格弹窗选择
- 规格值录入交互已优化，不再强依赖英文逗号

### 5.2 商城用户能力

- 新增商城用户表 `mall_user`
- 商城用户后台页、详情页、运营记录页已补齐
- 会员定义已从“微信用户”转为“绑定手机号的商城用户”
- 小程序已支持资料页、手机号绑定、会员身份展示

### 5.3 订单售后能力

- 订单详情和订单列表已支持更多售后信息展示
- 新增售后工作台
- 小程序已支持售后中心
- 客服动作、审核动作、退款结果已补操作留痕

### 5.4 客服能力

- 已确定采用微信原生客服
- 首页新增客服入口和咨询引导
- 不在底部 tab 中新增客服 tab

### 5.5 小程序稳定性

- 页面首批依赖登录态的接口，统一在 `app.initPage().then(() => {})` 之后调用
- 小程序主包已清理未使用的大图资源，当前资源结构可继续维护

## 6. 当前代码约定

### 6.1 小程序接口调用约定

依赖登录态的接口，页面中统一这样写：

```js
app.initPage()
  .then(() => {
    // 接口调用
  })
```

如果一个页面首屏有多个接口：

```js
app.initPage()
  .then(() => {
    this.loadMallUser()
    this.orderCountAll()
    this.loadAfterSaleStats()
  })
```

说明：

- 不建议在 `api.js` 再额外包统一登录守卫
- 关键是“先登录，再请求”
- 如果 `onLoad` 和 `onShow` 都会触发请求，建议做页面级异步复用

### 6.2 二开范围约定

当前业务代码主要集中在以下模块：

- `joolun-wx/joolun-mall`
- `joolun-wx/joolun-weixin`
- `joolun-wx-ui`
- `joolun-wx-ma`

若依原生模块原则上不做无关改动。
